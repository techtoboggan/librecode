Why LibreCode

What makes this different

We forked because the tools we depend on should respect the ecosystems we build in.

Real Packaging

COPR for Fedora/RHEL. AUR for Arch. Nix flake. Not a static .deb download from a GitHub release page. Your package manager should manage your packages.

Agentic SDLC

A formalized agent loop with clear instruction compilation, permission boundaries, and tool capability declarations. Not a prompt wrapped in a while loop.

Community First

No vendor lock-in. No cloud dependencies for core functionality. CI/CD rebuilt from scratch. Open development, open roadmap.

AI tools should be free software

The AI coding assistant you use every day shouldn't be a black box with a monthly bill and a terms-of-service that changes quarterly. It should be something you can inspect, modify, package, and distribute.

LibreCode is a fork of opencode with one goal: take a solid foundation and make it into the tool we actually want to use. Properly packaged, properly structured, properly open.

We believe the best developer tools come from developers who use them. That means real Linux packaging, clean architecture, and no artificial limitations.

Principles

How we build

01

Package managers exist for a reason

Every release goes to COPR, AUR, and Nix. Static binary downloads are a fallback, not the plan.

02

Architecture over accumulation

Decompose megafiles. Formalize the agent loop. Single-responsibility modules. No 67,000-line prompt files.

03

Enterprise without the enterprise tax

Permission systems, audit logging, MCP server management, session branching — built into the open source core.

04

Provider agnostic

20+ LLM providers out of the box. Local models welcome. No single-vendor dependency. Your keys, your choice.
